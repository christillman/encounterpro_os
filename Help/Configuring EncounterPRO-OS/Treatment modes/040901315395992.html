Epro uses wrong treatment workplan after upgrade (ex: PPD service bars for vaccines)

Cause:

Mod level 141 requires that there be a treatment mode whose treatment workplan is the same as the default treatment type workplan.

Example:  the Vaccine treatment type's Version 4 default treatment workplan is "Vaccine with Admin Codes" but the only mode on the treatment mode tab is PPD.  After the database's upgrade to mod level 141, when a vaccine is ordered even with no treatment mode, it will use the PPD treatment mode's workplan instead of the stated default.  In this case, look to see what the default treatment workplan is, create a treatment mode (the mode needs <=24 characters) and attach that default treatment workplan to it.

If you are working in version 5, go ahead and set the default treatment modes per office.


Treatment mode naming conventions:

  • If the workplan has a "close treatment" service at the end of it, start the name with Enc
  • If the treatment stays open after the workplan is finished, start with Ong (not necessary with meds!)
  • Give a short name or abbreviation for the main service or services in the workplan
  • Describe when the service happens
  • Describe who does it (if not the workplan owner / person who ordered the treatment)

Examples:  Ong Drilldown Now, Enc Drilldown by Staff, Enc Coll and Perform, Enc Dashboard, Enc Dashboard Now, Medication No Print, Medication Print Now, Ong Send or Refer Out, Ong Drilldown by Staff, Ong Inoffice Staff, Ong Inbox Admin

Solution:

Create a treamtent mode for each treatment type that uses the default treatment type's workplan.
