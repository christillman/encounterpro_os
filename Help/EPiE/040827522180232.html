How to: Use the EPiE Agent 1.0

Solution:

EpIEAgent allows EHR interface partners to upload data intended for import into the EncounterPRO database or download data sent from an EncounterPRO database for a mutual customer without working directly with the EpIEGateway SOAP service.  It resides on the interface partner's network and communicates with the EpIEGateway SOAP service at EHR headquarters.

 

EpIEAgent is intended for use on the Microsoft Windows platform, but it is possible to run EpIEAgent in a Unix/Linux environment using the Mono framework.  However, EHR will provide no support for running EpIEAgent on Mono.

 

The latest version of EPIE Agent is found in EHR FTP site under "EP_Upgrades[Production or Beta]EPIE Agent". After EpIEAgent is installed, it must be configured.  This can be done either by running the "Configure Feeds" shortcut created in the EpIEAgent program group, or by running the EpIEAgent.exe executable with the -c argument from the command line.  The configuration screen looks like this.

 

 

  1. The first thing to do is set the general configuration options. To do this, Select "General Options" from the File menu.



  2. The three pieces of information on this screen are the Interface Partner's Owner ID, EpIE User ID, and EpIE Password. All three are provided by EHR. Without providing this information, no messages can be sent to or received from the EpIEGateway SOAP service.

    Each path by which messages travel to or from the interface partner is called a "feed" in EpIEAgent. There are currently two types of feeds supported: File and HL7 Socket. To create a new feed, click the "New" button on the main configuration screen.


Here's a description of each of the options you see here:

Enabled - Determines whether this feed operates when EpIEAgent runs.

Upload - Determines whether this feed will upload data from the interface partner to EpIEGateway

Download - Determines whether this feed will download data from the EpIEGateway to the interface partner

Description - This description is displayed in the dropdown menu of available feeds and is also included in any error message descriptions.  This field is required.

AddresseeId - This is the Interface Service ID of the interface partner and this is found by looking up in CMA under the partner->interface service page.  This field is required.

CustomerId - This is an identifier the customer to send data to or receive data from.  For uploading, it can be an identifier defined by the interface partner or the Customer ID defined by JMJ.  For downloading, the JMJ-defined Customer ID must be used.  This field is required.

CustomerId is a JMJ AddresseeId - If the CustomerId used is the ID defined by JMJ, check this box so that EHR's integration engine knows that no customer identifier mapping is necessary.  This field is required.

DocumentType - The type of document consumed by or sent by the interface partner.  This value is found in CMA by looking up under the partner -> Interface Service.  This field is required

Class - This determines the type of feed to use.  The options are FileWorker and HL7SocketWorker.  This field is required.

 

Once a Class is selected, more options specific to that type of feed will appear.

 

All Feeds

File Feeds

HL7 Socket Feeds

CopySuccess - You may provide a folder path in this field in which successfully uploaded or downloaded messages will be written.  The files will be named with a sortable date-time string and have a txt extension.

Folder - The folder in which to find messages for upload and/or to write messages that are downloaded.  This field is required.

ListenAddress - The IP Address or DNS name of the listening interface on which HL7 messages will be received before upload to the EpIEGateway SOAP service.  If this is left blank, the feed will listen on all available interfaces.

CopyError - You may provide a folder path in this field in which messages will be written that had an error in uploading or downloading.  Downloaded messages will only appear in this folder if there was an error after the message was successfully downloaded from the EpIEGateway SOAP service.  The files will be named with a sortable date-time string and have a txt extension.

ReadFilter - The filename pattern to filter with when reading messages for upload (for example, *.txt or *.*).  This field is required for upload.

ListenPort - The TCP port on which the feed should listen for HL7 messages.  This field is required for upload.

 

OutputName - The filename pattern to use when writing downloaded messages to files (for example, *.msg).  Asterisks will be replaced with GUIDs.  This field is required for download.

SendAddress - The IP Address or DNS name of the interface to send to when downloading messages from the EpIEGateway SOAP service. This field is required for download.

 

 

SendPort - The TCP port to send to when downloading messages from the EpIEGateway SOAP service.  This field is required for download.

 

 

HL7Header - A comma-delimited list of hex-encoded characters expected at the beginning of an HL7 Socket transmission.  If this is left blank, the default is "0x0b".

 

 

HL7Trailer - A comma-delimited list of hex-encoded characters expected at the end of an HL7 Socket transmission.  If this is left blank, the default is "0x1c,0x0d".
