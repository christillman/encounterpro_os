How to: Troubleshoot Encounter Billing Problems.

Solution:

This solution lists several common problems with encounter billing and how to correct those problems.

Details:

I'm finished with my encounter, but there is nothing showing up in the billing box on the Exit tab.

Possible causes:

  • There is not a valid assessment for the encounter. Be sure that you have ordered a new assessment or updated an existing assessment.
  • The encounter type is not billable.

The lab I ordered is attached to the wrong assessment.

Possible causes:

  • The lab was not ordered from the treatment list for the appropriate assessment. The treatment can be associated with the correct assessment. See How to: Associate Treatments With a Specific Assessment.

The encounter level suggested by encounterPRO is too low.

Possible causes:

  • There is no HPI or ROS documented.
  • The decision-making is more complicated than the default Minimal level
  • Review how the billing level was calculated. See How to: Review the Calculated Encounter Visit Level.

The encounter level suggested by encounterPRO is too high.

Possible causes:

  • Review how the billing level was calculated. See How to: Review the Calculated Encounter Visit Level.

The wrong assessment is showing as the primary assessment for the encounter.

Possible causes:

  • The system defaults to listing the first assessment chosen during the encounter as the primary assessment.
  • Assessments can be re-ordered on the Patient Chart and in the billing. See How to: Rearrange Assessments on the Patient Chart.

I have a well encounter that also has a sick assessment. How do I add the -25 modifier?

Possible causes:

  • The sick visit needs to be added as a secondary visit code. See How to: Document a Well Visit With a Sick Assessment.
