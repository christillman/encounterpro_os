Purpose

This solution explains the process that encounterPRO uses when electronic refill requests are submitted via SureScripts-RxHub. There are several stages to the process:

  1. e-Refill request is received
  2. e-Refill request is matched to a patient and existing prescription medication treatment. If encounterPRO cannot match the request to a specific patient and treatment, the request goes to the To Be Posted list.
  3. encounterPRO sends a Task to the provider who last ordered the medication treatment to approve or deny the refill request.
  4. The provider can:
    • Approve the refill request 'as is'
    • Deny the request
    • Approve the request, but with a modified prescription
    • Approve the request, but with a change in the medication

Before You Begin

Typically your practice will begin receiving refill requests electronically 4 to 6 weeks after you begin sending new prescriptions via SureScripts-RxHub. Prescriptions that originated in the SureScripts-RxHub system will be eligible for an electronic refill request. To read more about the SureScripts-RxHub process, go to the SureScripts-RxHub website.

Handling e-Refill Requests for Controlled Substances

If the medication in the refill request is a controlled substance, the provider can select 'Approve' but the system will display a visual reminder that the prescription must be printed and signed.  When the user selects Finished the system automatically displays a message notifying the user that this refill WILL NOT be sent electronically.  EncounterPRO sends 'Denied-New RX to Follow' as refill response to the requestor.
The ordering provider has the choice of:

1)      Displaying the Document Management screen immediately.

2)      Done (don't print).

How encounterPRO Matches the e-Refill Request

When the pharmacy submits a refill request via SureScripts, encounterPRO will attempt to match the patient and specific prescription with an outstanding treatment for that patient. At this point, one of several things will happen:

A e-Refill Request is Received... Next... Then...
encounterPRO finds a matching patient encounterPRO finds a matching prescription medication order
§  A Task is sent to the user who ordered the lab to review the lab results
encounterPRO finds a matching patient encounterPRO does not find a matching prescription medication order §  The request is saved to the To Be Posted list under Utilities, Scan/Post Images 
§  The user must manually post the request to the correct patient and send a Task to the ordering provider
encounterPRO does not find a matching patient §  The request is saved to the To Be Posted list under Utilities, Scan/Post Images  
§  The user must manually post the request to a specific patient and send a Task to the ordering provider

When a matching patient and prescription cannot be found, the request must be handled manually. See How to: Manually Process an e-Refill Request.

How to Handle the Refill Request Task

After the e-Refill Request has been successfully matched to a patient and specific prescription medication order, encounterPRO sends a Task to the ordering provider. Follow these steps to complete the refill request:

  1. From the Office View, select Tasks. The Task list for the current user appears.
  2. Select the e-Refill Request bar. The Medication Refill Request screen appears.
  3. Review the refill request at the top of the screen.
  4. If needed, select the View Chart button at the bottom of the screen to access the patient's chart. When finished reviewing the chart, select Exit.
  5. At this point, the provider can do one of the following:
    • Approve the request
    • Deny the request
    • Approve with Modifications
    • Approve with a Different Drug
  6. If you want to approve the refill 'as is' select Approve, enter a Refill Comment  and select I'm Finished. The response is sent to the pharmacy and the patient's chart is updated with the date of the refill and the refill comment.
  7. If you want to Deny the request, select Denied, enter a Refill Comment or select a Reason Code, then select I'm Finished. The response is sent to the pharmacy. A progress note is saved in the patient's chart that the refill request was denied.
  8. If you want to approve the request with changes, select Modify RX then I'm Finished. The Prescription Drug Ordering screen appears. Make the needed changes to the prescription, then select Finished. The Document Manager screen appears. At this point, the provider can select Send to send the updated prescription back to the same pharmacy. If a different recipient or route is needed, the provider can select it on this screen. Select OK when finished with this document. The patient's chart is updated with the date of the refill, the modifications made, and any refill comment.
  9. If you want to approve the request with a new medication, select Change Med, then I'm Finished. The Prescription Drug Ordering screen appears. Make the needed changes to the prescription, then select Finished. The Document Manager screen appears. At this point, the provider can select Send to send the updated prescription back to the same pharmacy. If a different recipient or route is needed, the provider can select it on this screen. Select OK when finished with this document. The patient's chart is updated showing the previous prescription as closed and opening a new prescription treatment with the new drug.
