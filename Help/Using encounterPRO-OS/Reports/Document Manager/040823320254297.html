Description

When a document has been ordered, it will appear on the Document Management screen. The screen can be accessed from the Patient Chart, from the treatment room, or from certain menus. Depending on the button used to access the screen, the screen will display the documents for that treatment only, all documents for the current encounter, or all documents for that patient.

Choosing the Recipient
The recipients available depend on the document purpose, document configuration, and the recipient/routes purchased.
encounterPRO will default the recipient based on the following rules:

  • If a document of this specific type (context + object) has been previously ordered, that recipient defaults. A specific type would be a prescription for the same drug.
  • If no document of this purpose was previously ordered, then the recipient designated as the Primary for this recipient type/specialty on the Care Team list defaults. A purpose could be NewRX or Referral letter.
  • If no Primary recipient has been designated, then the system picks the first recipient (top) on the Care Team of the specified recipient type.

If the default recepient is not the desired recipient, then the user can select the recipient bar and select from the available recipient types.

Choosing the Route
After the recipient is selected, the system displays the routes (channels) available. Common routes include:

  • Printer
  • Secure Email *
  • Fax
  • E-prescribing *
  • Mail ("snail" mail) *

* Routes will be made available as subscription services, such as e-prescribing via SureScripts-RxHub.

Viewing the Document
If you want to review the document before sending it, select the View button for that document.

Status of the Document
The Status bar shows the current state of the document:

  • Ordered - most documents will show a Status of Ordered when the Document Management screen is first displayed. If the document was configured to Print Now, then the Status of Ordered will not be visible.
  • Created - if the user Viewed or Edited the document, the Status will show as Created. The document has not been dispatched yet.
  • Sent/Printed - depending on the route selected, the Status will be Sent or Printed. If a document is configured to Print Now, the Status will show as Sent or Printed automatically very soon after the document was ordered.
  • Completed - if the document was sent electronically, encounterPRO can receive messages indicating that the document was received. The Status will show as Complete. Once the Status shows as Completed, the document cannot be re-sent or edited.
  • Cancelled

Other Options
There are several other options you can perform on a document:

  • Edit - displays the dashboard for the treatment or object that the document pertains to
  • Re-send - if the document was not sent successfully, the user can choose to re-send it
  • Cancel - if the document was not yet sent or was not sent successfully, the user can choose to cancel it. (If the document has a status of Completed, it cannot be canceled here).

Sending the Document
When you are satisfied with the document, the recipient and route have been chosen then the document can be dispatched to the printer or electronic route. The button text will show Print or Send, depending on the route selected. Select the Send or Print button.

Details:

How to Access This Screen

  • To view documents only for a specific treatment select the icon next to that treatment on the Patient Chart
  • To view all documents for a specific encounter, do one of the following:
    • Select the icon next to that encounter on the Patient Chart
    • Select the Document Manager button on the menu from the exam room or Checkout room
    • Select the icon next to the name bar in the Office View
  • To view all documents for a patient, select the Document Manager button on the Patient Demographics screen.
