Purpose
There are certain instances when the user will be prompted to dispatch a workplan. Workplans can be kicked-off when:
  • A patient checks into the office who has a follow-up visit ordered
  • The user scans or imports a document into a folder that has an optional workplan attached

If a workplan is pending for a patient, the Dispatch Workplan screen will appear. It will

Details:

Steps
Follow the steps below:
To Dispatch a Patient Workplan:
When a patient is checked into the office who has follow-up orders pending the user has the option to launch the patient workplan or not.

  1. When the patient checks into the office, the New Encounter Appointment/Followup screen appears.
  2. You have two options:
    • If there are multiple follow-ups listed, select the appointment and follow-up that is related to this encounter, then select OK.
    • If today?s encounter is not related to any of the pending appointments or follow-ups, select This encounter is not related to a followup order.
  3. If you selected a follow-up in step 2, the follow-up treatments will be automatically ordered.

To Dispatch a Folder Workplan:
When a document is scanned into or imported into a folder with an optional workplan attached, the user can launch the workplan or not.

  1. When the user posts or imports a document into the folder, the Attachment Workplan screen appears. The name of the workplan and the default workplan owner are listed.
  2. You have two options:
    • If you want to dispatch the workplan, confirm the user or role to whom the system will assign the workplan tasks, then select Finished.
    • If you do not want to dispatch the workplan listed, select No Workplan at This Time. The document is posted to the folder.
