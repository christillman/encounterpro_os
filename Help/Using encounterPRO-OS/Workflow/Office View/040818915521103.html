Description
Service bars are often created when a service is ordered as part of a workplan. Services are one of the main building blocks in workplans allowing users or the system to perform certain operations.

When a user selects a service bar from the Office View, a new screen displays. If the user does not have the authority to perform the service, a message will appear.

Service bars can be in the color of a specific user or a role, depending on whether that service is 'owned' by a specific user or is currently available for any user with that role. Once a user selects a service bar, the system shows the service bar in that user's color. If the user is unable to complete the service, he/she has the option of keeping ownership of the service or leaving it for the role.



If the user selects anywhere in the treatment room on the Office View, that room is displayed. The service bars for the patient in that room are displayed in a larger view.

Each service bar has a brief description of the service that has been ordered, such as Administer Varicella and the number of minutes that service has been waiting. The color will indicate the specific user (generally the provider) or the role for whom that the service was ordered.
