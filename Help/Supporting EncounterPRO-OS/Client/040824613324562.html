Table Modification: Configure the Vaccine Consent Statement directly in the tables

Solution: First, review Solution # 040823718241845, 'How to: Configure the Vaccine Consent Statement'. If you are unable to update the statement using the interface, you may do so directly in the tables.
Details:

You can add a preference directly to the table 'p_preferences' (Unfortunately, it is currently limited to 255 characters.) :

Preference_type = ?PREFERENCES?
Preference_id = ?Vaccine Consent Statement?



You can display additional text (1,024 characters) by going directly into the tables. Add the following information into 'o_Service_Attribute'

  • service = VACCINE_SIGNATURE
  • attribute = Vaccine Consent Statement
  • value = TYPE YOUR TEXT HERE!



NOTE: In config 19, mod level 137 the following logic is introduced: Added logic to look for 'Vaccine Consent Statement' service attribute before preference. If attribute exists (up to 1024 characters) then it is used. If attribute does not exist, then preference is used.
