How to: Set the default encounter type for PM Integrations

Solution: This article describes how to set the default encounter type for practices using a Practice Management integration software. The default encounter type for non-hl7 interfaces is set as 'SICK'. If you need to change this, refer to the table. For HL7 interface, it is set up in 'o_Preference' table.
Details:

This article describes how to set the default encounter type for practices using a Practice Management integration software.

Resolution
The default encounter type for non-hl7 interfaces [like medisys,medical manager,fox meadows,lytec,medisoft] is set as 'SICK'. If you need to change this to some thing else, then add or edit the following row in c_component_attribute table.

 component_id  attribute_sequence  attribute  value
 MSGHANDLERSERVER  1000079  default_encounter_type  enter name of encounter type

For HL7 interface, it's set up in 'o_Preference' table as preference_id of Default_Encounter_type'.

Affected Products
Product       PM Integration
Version       All Versions
