Troubleshooting Lab Integration: Problems with Tasks

Cause: Both problems are related to the default treatment workplan.
Solution: Practices that are utilizing lab integration via .xml file format, need to set default treatment workplan for labs to Lab Ongoing Clinical Staff Collect or Send/Refer Out.
Details:

Practices that are utilizing lab integration via .xml file format, need to set the default treatment workplan for labs to: 
Lab Ongoing Clinical Staff Collect or Send/Refer Out (Integration)
NOTE: Wait service must be configured to include:
Progress Type= Results
Progress Key= Status
Also, the Wait service can be configured to include: Progress Type= Results and Progress Key= Status by running this script:
Insert into c_workplan_item_attribute (workplan_id, item_number, attribute, value)
 Values ('2192','8134','wait_progress_type','results')
go
Insert into c_workplan_item_attribute (workplan_id, item_number, attribute, value)
 Values ('2192','8134','wait_progress_key','Status')
go

Symptoms: Manual post of lab results does not yield 'send to-do' service. Automatic posting of labs does not send to-do item to appropriate physician.
