File not found on scan/post screen

Cause: This happens when a workstation does not have correct rights to the temp folder and CPRFILES.
Solution: All users need read and write access to c:\temp. Ensure CPRFILES have read and write for all. For a fat client ensure they have c:\temp. Set up weekly job on scanning PC and server to clear JMJ* files from temp .
Details:

Follow the below steps:

1) Ensure that all users have read and write access to c:\temp on database server and application server.
2) Ensure that CPRFILES have read and write for all.
3) If workstation is a fat client ensure that they have a c:\temp.
4) Ensure that a weekly job is set up on the scanning PC and the server to clear all JMJ* files from the temp folder.
