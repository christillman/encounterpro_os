How to: Make changes to the Prescription

Solution: This solution gives details for making the following changes to the prescription. Change from a 2-signature line to 1-signature line prescription. Change the prescription heading. Change the font. Move the Patient Name/Address and Drug Prescription up or down. Change the left and right printer margins. Remove the black line under the address. Change the height of the office address. Change the height of the footer. Change the footer control (text phrase) height for single signature line Rx. Remove the footer. Remove the patient address. Remove the Address. Remove the Signature. Remove the Bottom Black Line below the Signature. Remove the RX icon.
Details:

Note the use of the %SIGNING_PROVIDER_LICENSE% token. EncounterPRO will replace this token at runtime with the name and license number of the provider who owns or ordered the prescription. Other supported tokens for this field are:

Token

Substitution Value

Example

%SIGNING_PROVIDER%

Name of prescription owner

Familia, George K. MD

%SIGNING_PROVIDER_LICENSE%

Name and license number of prescription owner

Familia, George K. MD, Lic# 1234567

%SIGNING_PROVIDER_NPI%

Name and NPI of prescription owner

Familia, George K. MD, NPI: 9876543

%SIGNING_PROVIDER_LICENSE_NPI%

Name, license number, and NPI of prescription owner

Familia, George K. MD, Lic#1234567, NPI: 9876543


The NPI tokens require EncounterPRO Client 35.2 (or later) and EncounterPRO Server 6.1.2 (or later)

NOTES: EncounterPRO can get object attributes from many tables. In order to implement a policy of overriding general attributes with attributes from a more specific context (like office attributes overriding general attributes), each report will only 'see' the last attribute value for any attribute name. Put another way, if there are two or more attributes with the attribute name 'dwmodify', EncounterPRO will only recognize the last one and will ignore all others. In order to get EncounterPRO to recognize multiple datawindow modifier commands, the attribute names must all be different but must all begin with the characters 'dwmodify'. See above how, in order to get the reports to recognize multiple 'dwmodify' attributes, a number is sometimes appended to the 'dwmodify'.
