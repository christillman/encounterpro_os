How to: Configuring the Daily Lab Results Report

Solution: Configuring the daily lab results report requires three main steps. First, add a 'Daily Lab Results' button to 'Utilities' menu. Next, fix the 'Referrals By Day' TODO bug. Finally, default the Daily Lab Results screen.
Details:

Adding a Daily Lab Results button to Utilities menu
To add the button to the Utilities menu, follow these steps:
1) Click 'Utilities' and select 'Standard config menu default '.
2) Click 'Create local copy (if not already there)' and select 'Edit menu'.
3) Click 'New', select 'Service' and click 'General'.
4) Select 'Referrals by day' and click 'Finish'.
5) Click 'Change' button title to 'Daily Lab Results' and select 'Finished'.
6) Click 'OK', select 'Finished and click 'Exit'.

Fixing the 'Referrals By Day' TODO bug
NOTE: This has yet to be approved by Development but it does work.
If practice is not on version 35.7 or higher and clicks on the 'Daily Lab Results' button, it will always send a Not In Office TODO titled 'Referrals By Day'. This can be fixed by doing the following:

1) While logged into practice site, open the microsoft access db through 'Start', select 'Programs' and click 'MS Access'.
2) Open the most current cpr.mdb practice has then open the dbo_o_service table and filter service column for 'referrals by day' and change component id from svc_generic to SVC_UTILITY and then click mouse in a box below which will save work.

NOTE: Never click 'YES" when asked if you want to save changes to the design of table.

Default the Daily Lab Results Screen
If the practice wishes to have the 'Daily Lab Results' screen (from Utilities) default to a particular lab once clicked, follow the below steps. (Hint: Look up the consultant id in table dbo_c_consultant, filter specialty id column for '$lab' prior).
 
1) While logged into practice site, open the microsoft access db through 'Start', select 'Programs', and click 'MS Access'.
2) You will choose the 'cpr mdb' unless there is a 'cpr_40 mdb' then open dbo_o_service_attribute and filter service column for 'referrals by day' then confgure to appear as follows changing Labname to lab company's consultant id you looked up prior. (ie LAB_CORP)

dbo_o_service_attribute

service attribute_sequence user_id attribute value
referrals by day (autonumber)   specialty_id $lab
referrals by day (autonumber)   consultant_id Labname

Once completed click mouse in a box below table which will save work.

NOTE: If  the lab company does not default in the Daily Lab Results utility, double-check in dbo_c_consultant table to see if the consultant_id value is identical (for example, in some instances the value is 'Lab_corp' rather than 'Labcorp').
 
If  the lab company is not in the db as a consultant, you need to add it by performing the following:
1) Click Config, select 'specialties and consultants' and click 'laboratory'. 
2) If you do not see 'The Lab' on the list then click 'NEW' (on right), enter the name 'Labname' and complete info.
3) Click 'OK'.
