Database Config Releases

Solution Title: Database Config Releases
Overview:

Mark has created 2 new systems in DevIssues, both under the 'Config Master' category:

  • Config Classic
  • Peds Standard

The only release of Config Classic in DevIssues is 19.0, and corresponds to our classic Config 19.

The Peds Standard system has a 1.0.1 build in production, and a 1.1.1 build in development.  Please log any issues relating to further changes in Peds Standard to Devissues and assign the issues being addressed to the appropriate build.

I have created a repository in \\Techserv\Builds for the Config Master releases:

\\techserv\builds\Config Master\Config Classic
\\techserv\builds\Config Master\Peds Standard

This is where the database snapshot will be permanently stored each time a release is made.

Whoever needs to refer to the actual released files will need 'Read' access to \\techserv\builds.  No one outside of development should have 'Write' access to \\techserv\builds.

Ed, you are in charge of organizing the FTP site and any other locations that you want to contain copies of files from \\techserv\builds.  I am in the habit of copying client and server releases to the EP_Upgrades folders when I release the client and server, but I would be happy to turn that over to you, or to formalize or document where copies should be placed when systems are released.


In Summary….

  • Devissues - tracks build status and issues addressed in each build
  • \\Techserv\Builds - repository for released files for every build of every system
  • FTP Site - contains, among other things, a copy of some files from \\techserv\builds needed for implementation and support
