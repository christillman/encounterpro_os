Triage Protocol: Prescriptions Not Printing

Triage Protocol: Triage Protocol: Prescriptions Not Printing
Overview:

Unable to print prescriptions from EncounterPRO:

q  Auto RX printing is not working

q  Manual RX printing is not working

q  No RX printing

Possible Causes

The most common cause of printing problems is that EncounterPRO Services are stopped. Another common cause is that something changed in the printer setup - a new printer was added, printers were moved, etc.  Other possible causes include: printer is off-line, no paper in printer trays. If none of these is the cause, this type of issue will usually need to be sent to Level 2 to resolve.

Symptoms:

Get Additional Information

q  Is the caller the only one having trouble printing prescriptions or is it all users?

q  Is the problem with Auto RX printing, manual RX printing, or Refill RX printing? Of all of the above?

q  What printer are you trying to print to?

Resolution:

Connect to Customer Site

Agent connects to site using Admin login. Stop and re-start EncounterPRO Services. Print a printer test page on the printer that they are having the problem with.

If the problem is with?

Then?

Auto RX not printing

Ask the caller if they can manually print RX. If so, then the printer is communicating with EncounterPRO.

Could the RXs be printing out somewhere else? Different printer than expected, did printer get moved?

Agent documents all information and steps performed and assign ticket to Level 2.

Manual RX not printing

Ask the caller if Auto RX printing is working

Agent documents all information and steps performed and assign ticket to Level 2.

Refill RX won't print

Agent documents all information and steps performed and assign ticket to Level 2.

Printer test page won't print

Ask caller to confirm that the printer is turned On and there is paper in the tray? Retry printing test page.

If it still does not print, agent documents all information and steps performed, assign ticket to Level 2.

Instruct the caller that this appears to be a Network problem. They need to contact their Network Support.
