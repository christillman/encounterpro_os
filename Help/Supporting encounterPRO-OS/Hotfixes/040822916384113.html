Lab Results are showing 'Ready' status in EPIE but not uploading to customer.

Cause: The lab results download engine is not enabled or is having some issues.
Solution: Check to see the encounterpro services are running. Check the 'Event Viewer' to see any errors logged under 'EPROSCHEDULE' / 'EPROLIBNET'.
Details:

1) Check to see the encounterpro services are running.
2) Check the 'Event Viewer' to see any errors logged under 'EPROSCHEDULE' / 'EPROLIBNET'. If yes then it could be one of the following:
  a) If there is EPROLIBNET error that shows the following error text: (Like Validation Failed)
    1] Then Rerun HOTFIX 99 SecureGateway and Go to 'EncounterPRO', select 'Configuration' and click 'Preferences'.
    2] Set the 'EPIE Username' and 'EPIE Password'.
    3] You don't need to rerun HOTFIX 99 but for very few cases I noticed something make the EPIE Username encrypted and noticed below errors. But if you get validation failed then try reentering the EPIE username / pwd and restart the EncounterPRO Service to see it works.

<EncounterPROContext cpr_id='''' PrimaryObjectKey='0'
PrimaryContextObject=''General''><ContextObject ObjectKey=''General''
ContextObject=''General''><ContextProperty name=''customer_id'
unit=''''>2094</ContextProperty><ContextProperty name=''database_version'' unit=''''>05</ContextProperty><ContextProperty name=''major_release''
unit=''''>4</ContextProperty><ContextProperty name=''modification_level''
unit=''''>127</ContextProperty><ContextProperty name=''provider'' unit=''''>JMJ Service</ContextProperty></ContextObject></EncounterPROContext>

System.Web.Services.Protocols.SoapException: Server was unable to process request. ---> Validation failed.

  b) For some cases in very few customers it was throwing 'EPROLIBNET' with 'Error calling setcallbackwindow()'. This case it is recommended to reinstall EPROLIBNET to fix this kinds of issues.

4.05.9.15 (6/13/2007) <Server PB Code> >>> u_component_observation_jmj_com - (xx_initialize()) Error setting callback window (Error calling external object function setcallbackwindow at line 81 in function xx_initialize of object u_component_observation_jmj_com.)

3) If encounterpro services are running and none of the above errors noticed then it might be the result engine is not setup or disabled.
4) Go to 'DB Maintenance', select 'Utility' tab and run utility 'Enable the lab results interface'.
5) By default it's 2 minutes to download the results. After doing step b and after 2 minutes if you still notice the messages waiting in 'Ready' status then send it to 'Integration Specialist' for advise.

NOTES: It is not EPIE issue. If you notice lot of messages waiting in 'Ready' status that means the encounterpro service at customer site is either not running or having issues to download the results. Resending the messages to this case is not a solution. Because the messages are successfully processed and ready for customer to download it. Just make sure the customer is in 5.1.8 (with 5.1.9 files) or any later versions. If they are but still having this issues then notify 'Development:'
