How to: Increase access code (password) size (v2, v4)

Solution: For version 4, change access code in 'o_preferences' table. Under 'o_preferences', filter in column 'preference_id', for 'password_length', and change access code length. For version 2, change access code in the interface. Go to 'Configuration', select 'Preference level' as 'Global', and change access code length to 6. Add extra zeros at the end of the access codes. To change user access code, go to 'c_user' table.
Details:

Resolution
For version 4:
The access code can be changed in the 'o_preferences' table.

Under 'o_preferences', you would filter in column 'preference_id', for 'password_length', and then change the access code length to whatever you want.

After changing the length, you should add extra zeros at the end (or beginning) of the access codes according to how much you increased it. This way, the user can log on using their current access code + zeros. To change user access code, go to the 'c_user' table.

Ex.- increased access code to 6 digits - original: 9876, after change: 987600

For version 2:
The access code must be changed in the interface.
Go to 'Configuration', select 'Preference level' as 'Global', and change access code length to 6.

After changing the length, you should add extra zeros at the end (or beginning) of the access codes according to how much you increased it. This way ,the user can log on using their current access code + zeros. To change user access code, go to the 'c_user' table.
