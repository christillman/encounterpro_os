Problem: Address2 Field Populated with City Name

Cause: Typically this miss-mapping occurs as a result of a known problem found in EP Server version 5.x.x
Solution:
Details:

Resolution
Upgrade the practice to EP server version 6.x.x or higher. This will resolve the issue for all patient demographics from the time of the upgrade and going forward.

However the fix is not retroactive and in order to affect the repair for existing records the following script(s) must be run to move the city name from the ?Address2? field to its proper location.


This first query will Check that the data in address line 2 is the city field:

select billing_id,address_line_1,address_line_2,city from p_patient where address_line_2 is not null

This query will remove city from address line 2 and place it in the city field:

update p_patient
set address_line_2=null, city = address_line_2
from p_patient where address_line_2 is not null
