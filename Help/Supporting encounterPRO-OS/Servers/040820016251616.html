Solution Path: Supporting encounterPRO-OS > Servers

Problem: Duplicate Services with encounterPRO Service

Cause: Service threads are not terminated while shutting down encounterpro service.

This only applies to vs5 series of eproserver and was fixed in eproserver vs6.
Solution:
  1. Stop the EncounterPRO Service. Go to Task Manager and see whether any orphan 'eproserver.exe' are still running. (If not that is fine.) If there are orphans running they should be killed automatically by installing the following.
  2. Copy the EncounterPRO Server Program files (usu found on at D:\Program Files\JMJ\EncounterPRO Server); Paste and rename the folder OrgEncounterPRO Server. (we perform this step just in case we have to restore the original server files).
  3. Go to Ftpsite and copy the files under EP_Integrations\Version 4.0\EproService into the 'EncounterPRO server' Program Files folder (when prompted, replace all three files)
  4. Start the EncounterPRO Service. This should kill all the orphan threads before spawning any new instances.
  5. Make sure all the services are working fine:
    1. Launch Epro Agent Status (EncounterPROService Monitor) 
    2. Verify services running equal amount of threads in task manager
    3. Stop and restart the services to make sure all threads successfully stop
Details:

Affected Products
Product: encounterPRO server 5.1.4
PM Integration, All Versions
