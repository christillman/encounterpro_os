How to: Set the Report Size for e-Prescribing

Solution:

Set Rx Report Size

This script will save the datawindow formatting commands in c_Report_Attribute and o_Report_Attribute and replace them with the proper formatting attributes for a PDF document 4.25' wide and 5.5' tall.

UNSET Rx Report Size

This script will delete the PDF formatting attributes and restore the attributes that were saved by the 'Set Rx Report Size' script.

The purpose of these new scripts is to separate the activate/deactivate operation from the report settings.  This way someone can test the Rx report prior to activation by running the 'Set Rx Report Size' script, then testing the rx report printing, and then running the 'UNSET Rx Report Size' script to restore the non-PDF settings.  Note that this should not be done while the users are actively printing prescriptions the old fashioned way, because setting the PDF attributes may mess up the rx when printed directly from the report service.

This Set/Unset strategy is only needed for the client 5.1.11.  Beginning with client 5.1.12, the PDF settings for the Rx report will automatically be set when the report is sent to the document manager so the testing can be done anytime, even while the users are still printing prescriptions the old fashioned way.
