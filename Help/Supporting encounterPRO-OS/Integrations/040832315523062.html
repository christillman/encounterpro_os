Triage Protocol: Patients Not Checking In

Triage Protocol: Triage Protocol: Patients Not Checking In
Overview:
Symptoms:

Patients are being checked in via the PMS but are not showing up in the EncounterPRO Waiting Room

Resolution:

Get Patient Examples

q  New patients are not checking in (get patient name and id number)

q  Existing patients are not checking in (get patient name and id number)

q  Neither new or existing patients are checking in (get patient name and id number)

Connect to Customer Site

Agent connects to site using non-Admin login.

If?

Then?

New patients cannot be checked in

Check that the patient ID assigned to the new patient does not already exist in the db.

If patient ID exists, inform caller that the new patient ID must be unique. They need to change it in the PMS and try checking in again.

Caller is using Version 2 and cannot check in existing patients

Check for any open encounters for the patient. Version 2 will not allow you to start a new encounter if there is an open encounter for that same patient.

If there is an open encounter, inform the caller that this was the problem and close the open encounter. Ask them to re-try checking in the patient.

If neither new or existing patients are checking in

Stop and re-start EncounterPRO Services.

Inform the caller that you have re-started the EncounterPRO Services. Ask if patients are checking in to the Waiting Room now?

If the patient(s) are still not properly checking into the Waiting Room, document all information, including the exact error message and all steps performed in the ticket and assigns to Level 2.
