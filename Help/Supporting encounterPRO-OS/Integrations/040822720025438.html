How to: Configuring the Lab Integration Workplan

Solution: Configuring the workplan for the lab integration requires three main steps. First, set up the default workplan for the Lab treatment type. Next, default the Collect Specimen Menu. FInally, assigning who receives the ToDo to Review the Lab Results.
Details:

NOTE: As of 4/9/2008, practice must be on min v4.34.7 w/EP Server 6.1.9 or v5.25 w/EP Server 7.0.10 for any kind of interface.

Set up the default workplan for the Lab treatment type
In EncounterPRO, set default treatment workplan (the script should add the workplan, the trainer must assign as the default).

1) While logged into practice's Epro icon go to 'Configuration' and select 'Treatment Type'
2) Click 'Lab', select 'Edit treatment', and click on the default workplan.
3) Do a search under description for 'Integration', in 'specialty' choose the workplan for 'Lab, Ongoing Clinical Staff Collect or Send/Refer Out (Integration).
4) Click 'OK' and finished.
NOTE: If this workplan does not appear, the practice may be on the wrong config level. Notify Support.

Default the Collect Specimen Menu
If the practice wishes to have the 'Patient Referred To:' or 'Specimen Sent To:' boxes in 'Collect Specimen Menu' default to a particular lab, see kb article #10640 Default the name of a lab company to appear automatically in the 'Collect or Refer Menu'.

Assigning who receives the ToDo to Review the Lab Results
Once a lab has posted to a patients chart a TODO item will be sent to the user or role assigned in the workplan. You need to assign a user or role the practice chooses, to the last step in the workplan you created above. Ask the practice who they prefer to receive 'review lab' TODO item when a lab is posted. A few common choices are as follows:

1) Practice prefers the doctor who ordered the lab to receive the TODO then assign the role of primary provider.
2) Practice wishes the encounter owner to receive the TODO then select the role of encounter owner.
NOTE: This could change frequently if any other kind of encounter is added after the lab is ordered. ie telephone encounter. To avoid this you can assign the role of workplan owner which will send the TODO to who created initial encounter.
3) A popular option, especially for a big practice, is to have you create a new role titled 'Lab Nurse' (or something similar). Remember to assign at least one appropriate user to the new role then that user can assign anyone else they choose. To create a role see kb article #10119 'Adding Users'.
NOTE: Not exactly accurate for adding 'roles' but best kb article I could find and works very similar.

To assign the user or role follow the below steps:

1) Click 'Configure' from the 'Office View'.
2) Select 'Workplans'.
3) Change the 'WorkplanType' to 'Treatment'.
4) Select the search option for 'Description' and type 'lab'.
5) Select the 'Specialty' button to display all matching workplans.
6) Select the workplan, 'Lab, ongoing clinical staff collect or send/refer out (Integration)'.
7) Select 'Display/edit' from the menu options.
8) Select 'Create local copy'.
9) Edit the workplan.
10) Select the 'Review lab step'.
11) Select 'Edit step'.
12) Select the bar for encounter owner.
13) Select the 'Ordered for' button.
14) Select  the user or role the client would prefer to receive the 'Review Lab' task.
