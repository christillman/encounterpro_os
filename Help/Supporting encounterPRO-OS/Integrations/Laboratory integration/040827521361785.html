Design Overview:

Quest is Uni-directional interface which receives the lab results from one or more lab vendors. EncounterPRO EPIE Quest Message handler receives the vendor lab results directly to EPIE thru webservice call between EHR Inc and Quest servers. It is then transformed into JMJDocument XML and downloaded by local EncounterPRO application. All the change requests and version managements for Quest are handled in EPIE interface and the configuration and route definitions are managed thru CMA. Go to CMA under Quest and see the list of clients in dial-up mode.

 

Transport Protocol

 

Quest sends the lab results to EHR Quest secured web services. In this model EHR directly works with the Quest in order to establish the secured connection. In this mode of transport all the messages goes thru EHR Clearing house where this requires Quest supply their unique Quest Account Number in order to distribute the messages to right customers. EHR Inc. already established a secured webservices between EHR server (EPIE machine) and Quest Customers.

Preparations

 

1.    Go to CMA and click on Interface Services and select the interface service type "Lab Results". If the interface service you are looking for is in the "Pending" list then it is not ready yet to install at Customers. If you could not find the interface service or that is not completed yet then contact EHR Development to find out the status on this.

2.    Check to make sure the EHR Client is in the recommended EncounterPRO versions above.

3.     Contact Quest project manager for the Customer so that EHR Inc. and Quest team work together in enabling the customer result messages from Quest thru webservice. Since all the customer lab results come thru single folder it is important to identify the Quest  "Quest Account Number" for the customer you are working with. Quest assigns "TEST CSID" and "PROD CSID" and gets these two from Quest rep in order to successfully subscribe the Quest interface.

Installation:

Installation / Configuration (@EPIE thru CMA)

 

1.    Subscribe the interface. (Epie regisration)

2.    Create the EPIE user account (epie account) for the EHR Client. These credentials authorize the EHR client to communicate with EPIE to exchange messages.

Training Schedule

 

Once the subscription is successfully completed schedule a training session and start working on the Installation / Configuration steps couple days prior to training session to make sure it is working.

Installation / Configuration (@Customer)

  1. Setup EPIE credentials. (Epie installation)
  2. Install the Lab Results Interface by going into EPROServer DbMaintenance Utilities->Enable LabResults Interface.
  3. Test the EPIE connectivity by sending test messages from CMA to EHR Client. See the screen shot below.


  1. Test the end-end connectivity by requesting the Quest to send sample lab results into the folder and make sure they showed up at the Customer Page at CMA. If the message did not come thru then click the "viewlog" to see if any errors were found. See the screen shot below
