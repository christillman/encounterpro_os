Error: Errors When Manually Posting Lab Results (Lab Integration)

Error Message: More than one patient identifier. Error processing document. User is kicked out of Epro while trying to post a document. User receives an error message after clicking 'Post' and/or is unable to post anything.
Cause: Not Applicable
Solution: For details on resolving the different error when manually posting, refer below.
Details:

'More than one patient identifier' message when posting results.
1) When a user selects results from scan/post to be posted, and clicks 'Post', she gets a message saying that Epro has found more than one patient identifier and to only click yes if you are absolutely sure that this is the patient you want.
2) ALWAYS click 'No'.
3) This happens because the lab company has sent over a patient id that is different from the Epro billing id.
4) Epro doesn't recognize this id, and if you click yes, a new patient chart will be created.
5) The lab company must either send over the Epro billing id, or no patient id at all.
6) In the absence of an id, Epro will select the patient by name, DOB, and SS#.

'Error processing document' message when posting results.
1) Many times when this happens the file is corrupted.
2) Delete the results, resend them and try to post them again or check to see that they were auto-posted to the chart.

User is kicked out of Epro while trying to post a document.
This is usually either a problem caused by the Citrix connection, in which case they may need an upgrade, or it is a matter of running all Office Updates (on server and fat clients).

User receives an error message after clicking 'Post' and/or is unable to post anything.
1) This is different than the issue discussed above. This occurs when user clicks 'post' in 'Scan/Post', gets an EPRO error message and is then unable to post any lab for any patient.
2) With Mod Level 128 and above, if they are having problems in posting the lab results (it throws error in posting XML document) try installing Hotfix 257 which fixes the following error.
