How to: Cancel an Encounter

Purpose
Cancelling an encounter deletes all information recorded during that encounter and deletes it from the patient record. Cancel an encounter, only if there was no encounter or if there was an inadvertent duplicate encounter created. If there is any valid information in the encounter, the user should Close the encounter to save that information and document that there was a partial encounter.

Before You Begin
Be sure that the encounter you are canceling has no data entered in it. Canceling the encounter means that the encounter was a mistake and no information entered in that encounter will be saved.

Details:

Steps
Follow the steps below:

  1. The best places to cancel a duplicate encounter are from the Waiting Room or the Checkout Room. Select the patient name bar, a menu of options appears.
  2. Select Cancel Encounter. A dialog prompts the user to enter a reason for canceling the encounter.
  3. Enter a reason or select from the list. The system prompts the user to be sure that they want to delete the encounter.
  4. Select Yes to delete the encounter.
    If there were unfinished services for the patient, the system prompts the user a second time to be sure that they want to delete the encounter.
  5. Select Yes. The encounter is cancelled. It can be seen only from the encounters tab of the Patient Demographics screen.
