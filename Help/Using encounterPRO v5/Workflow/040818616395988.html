Description
Workplans are the rules and steps used to implement workflow in the the encounterPRO system. There are several types of workplans:

  • Encounter workplans. These workplans detail the screens that will be presented, in which order, and for which users to guide the patient from the Waiting Room to the Checkout Room. An encounter workplan will be assigned to each Encounter Type, so that when the patient is checked in, a workplan is assigned and the system knows what steps need to be ordered.
  • Treatment workplans. Certain treatments are complex enough to require their own workplans. For example, when a provider orders a prescription drug, several screens and steps need to be performed by either the users or the system to produce a printed prescription. Treatments that require collection and results steps have been assigned a workplan to determine when a service bar is created, who is it assigned to, and if it is required to be completed during the current encounter. Treatment workplans are assigned to treatment types and specific treatments using Treatment Modes.
  • Folder workplans. When a document is saved into a folder, a workplan can be launched. Folder workplans can be used to notify a user when a lab result has been added to the system, to close an open treatment, etc. Folder workplans are especially important in Lab Integration. A folder can have only one workplan assigned.
