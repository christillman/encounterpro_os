How to: Run a trace for SQL 2005

Solution: You will find 'SQL Server Profiler'. Under Start > Program Files > MS SQL Server 2005 > Performance Tools > SQL Server Profiler
Details:

1) Open 'SQL Server Profiler'.
2) Select 'New Trace'.
3) Select the 'General' tab:
  a) Enter trace name.
  b) Save to file.
  c) Set maximum size to 50 MB
(*If running a performance trace, select 'Enable trace stop time' and set to run to the end of the business day.)
4) Select the 'Events' tab:
  a) Stored Procedures- RPC:Completed & Started
  b) TSQL- SQL:BatchCompleted & BatchStarted
5) To access all events if not shown by default, right-click the default event and click 'select event category'. That should display all events available to edit.
6) Then click 'Deselect event category' and individually reselect each individual event you need to monitor.
7) Click the 'Column Filters' button in the bottom right of the 'Events Selection' tab.

*For general performance issues, set a filter on the Duration: Equal to or greater than =1000. 
*For reproducable issues, set a filter on the SPID: Equal to (This can be found on the Server Status in VS 4.0 and Utilities Tab in VS 2.0)
