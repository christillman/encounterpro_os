Initial Version Dependencies

  1. EncounterPRO Client 4.35. 7
  2. Db mod level version must be 127 or later.
  3. Config is 4.05.19 and up
  4. EncounterPRO Server 6.1.9 or later
Design Overview:

EncounterPRO EPIE Lab Message handler receives the vendor lab results thru EPIE Agent from EHR client site(s) and transforms them into JMJDocument XML. The transformed messages in EPIE then downloaded by local EncounterPRO application. All the change requests and version managements for different lab vendors are handled in EPIE interface and the configuration and route definitions are managed thru CMA.  

 

TRANSPORT PROTOCOL

 

This selected lab vendor preferred to send the lab results to EHR Client site and writes them in a folder that is accessible to EncounterPRO EPIE Agent. This standalone installer at the EHR site (EPIE Agent under EHR FTP site) is listening for the lab results and sends them to EPIE thru EPIE Gateway.

 

Preparations

 

1.      Go to CMA and click on Interface Services and select the interface service type. If the interface service you are looking for is in the "Pending" list then it is not ready yet to install at Customers. If you could not find the interface service or that is not completed yet then contact EHR Development to find out the status on this.

2.      Check to make sure the EHR Client is in the recommended EncounterPRO versions above.

3.       Contact Lab vendor and get the file path and extension of the folder where these messages are created at EHR client site.

 

Installation/Configuratio:

Installation / Configuration (@EPIE thru CMA)

1.      Subscribe the interface. (EPIE Registration)

2.      Create the EPIE user account (EPIE Account) for the EHR Client. These credentials authorize the EHR client to communicate with EPIE to exchange messages.

Training Schedule

 

    Once the subscription is successfully completed schedule a training session and start working on the Installation / Configuration steps couple days prior to training session to make sure it is working.

 

Installation / Configuration (@Customer)

  1. Setup EPIE credentials. (EPIE installation)
  2. Install the Lab Results Interface by going into EPROServer DbMaintenance Utilities->Enable LabResults Interface.
  3. Install EPIE Agent at the EHR client to a machine where EncounterPRO Services are installed. This way it is easy to trace and migrating them during server switches.
  4. Test the EPIE connectivity by sending test messages from CMA to EHR Client. See the screen shot below.

 


  1. Test the end-end connectivity by requesting the vendor to send sample lab results into the folder and make sure they showed up at the Customer Page at CMA.
