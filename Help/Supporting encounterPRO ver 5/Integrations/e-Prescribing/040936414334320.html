Problem: Prescription Printing 8.5 x 11 After Installing Document Manager

Cause:

Is there a way to make the change to the pdf file so that it only displays/prints at 75% instead of the 190% it currently does?

As a test, printed rx from View screen of PDF file.  From view, the default view is 190%.  Changed that to 75% and selected printer from PDF file and printed.  The rx printed as should. 

Solution:

They shouldn't print from "View."  EncounterPRO can't keep a record of the printing if the print from "View".  They are trained to print by clicking the "Print" button.

Even if they do print from "View", the default zoom is a function of the PDF viewer program and the printer driver.  If they absolutely need to print from "View" then we say:

a) you shouldn't do that. 

b)  you need to make whatever changes are needed on the print dialog box yourselves.

Details:

Upon implementation of Document Manager, prescriptions now print full size on 8.5 x 11.  Mark spoke to Nathan and explained he would need to configure printer to print the rx 4.25 x 5.5.  I spoke to Nathan and he's under the impression he has the printer set up correctly.  Logged into Lansing and viewed printer "Nursesouth", where 8.5 x 11 rx was printed.  Printer settings were set to "Automatically Select" paper tray and paper was set to 8.5 x 11.  Tried to set paper size to 4.25 x 5.5 without success.  Spoke to Jake who set up virtual printer named "Nursesouth Rx" for testing purposes.  Was able to reset tray and reset paper size to 4.25 x 5.5.  Printed test rx - see second attachment for result.
