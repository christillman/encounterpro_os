Concept: Activate Document Management for E-Prescribing

Solution:

Activate Document Management

Document Management is an enhancement to general reporting which allows the EncounterPRO users to automatically persist the report as a permanent document in the patient's chart, and allows for the separation of the act of ordering the document from the act of choosing the recipient of the document.

When an EncounterPRO installation begins to use E-Prescribing, it is important that all of the workflows which create the actual physical prescription are updated to use document management.  The Prescription Report can be run from a menu or from a workplan, and since many EncounterPRO installations have customized configurations, the process to switch them all from traditional reporting to document management must be scripted as a set of criteria.

To accomplish this, the 'Activate Document Management' utility script in the DB Maintenance program will change the EPRO Report service to use the Order Document attribute for all TREATMENT menu items and workplan items EXCEPT where:

  • The service is ordered for Destination = SCREEN
  • The service does not have a specific report (user pick)
  • The workplan is attached to a Treatment Mode

Note that the utility script will preserve any previous setting for the Order Document  attribute, unless the service is specifically for the standard EncounterPRO Drug Prescription report or any report in the Drug Prescription report category.  This is done in an attempt to preserve any custom configuration that may already be in place redirecting specific reports to Document Management.

 

Deactivate Document Management

If, for any reason, the Document Management for E-Prescribing needs to be turned off after it has been turned on, there is a utility script in the DB Maintenance program called 'Deactivate Document Management'.  This script will remove the Order Document attribute for all TREATMENT menu items and workplan items that satisfy the same criteria as the Activate Document Management script.

Note that if there were any custom configurations already be in place redirecting specific reports to Document Management, the Deactivate Document Management script will remove them because it cannot tell the difference between a pre-existing Order Document attribute and one that was added by the Activate Document Management script.

Setting the Report Size for e-RX

Set Rx Report Size

This script will save the datawindow formatting commands in c_Report_Attribute and o_Report_Attribute and replace them with the proper formatting attributes for a PDF document 4.25" wide and 5.5" tall.

UNSET Rx Report Size

This script will delete the PDF formatting attributes and restore the attributes that were saved by the "Set Rx Report Size" script.

The purpose of these new scripts is to separate the activate/deactivate operation from the report settings.  This way someone can test the Rx report prior to activation by running the "Set Rx Report Size" script, then testing the rx report printing, and then running the "UNSET Rx Report Size" script to restore the non-PDF settings.  Note that this should not be done while the users are actively printing prescriptions the old fashioned way, because setting the PDF attributes may mess up the rx when printed directly from the report service.

This Set/Unset strategy is only needed for the client 5.1.11.  Beginning with client 5.1.12, the PDF settings for the Rx report will automatically be set when the report is sent to the document manager so the testing can be done anytime, even while the users are still printing prescriptions the old fashioned way.
