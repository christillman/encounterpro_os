Error: Importing Large Files

Error Message: u-component-observation-jmj-file - (get-attachments()) File too large -- or -- u-component-observation-jmj-file - (get-attachments()) address is required in server mode
Cause: The JMJ File Import external source component has several configuration options which control its behavior. If the configuration parameters are not appropriate to the situation, an error might occur.
Solution: These configuration parameters can be set globally in 'c_External_Source_Attribute', or office-by-office and computer-by-computer n 'o_External_Source_Attribute'. create a schedule record with your preferred interval and assign to #JMJ user.
Details:

These configuration parameters can be set globally in 'c_External_Source_Attribute', or office-by-office and computer-by-computer in 'o_External_Source_Attribute'.

The available configuration options setable in 'c_External_Source_Attribute' or 'o_External_Source_Attribute' (where external_source = JMJ_FILE) are:

attribute usage
max_files Sets the maximum number of files to import in a single pass. Defaults to 100.
max_file_size Sets the maximum size of a file to import in bytes. Default to 2,000,000.
move_to_success Sets the default directory for where to move the imported file if it is imported successfully. Defaults to Null which means don't move file.
move_to_error Sets the default directory for where to move the imported file if an error occurs during file import. Defaults to Null which means don't move file.
delete_file Set to 'Yes' to delete the file after it is successfully imported. Defaults to 'No'.
comment_title Sets a standard title for imported file. If not set, user may be prompted to supply a title.
xml_results Set to 'Yes' to interpret XML file as data instead of as an attachment. This is only useful in the context of importing treatment results with a treatment context present. Defaults to 'No'.
address Sets the file path to import files from. If not set in CLIENT mode then EncounterPRO will return without an error. If set to 'pick', EncounterPRO will prompt user for files to import. Must be set in server mode or error will result. If wildcard characters are included in file specification, EncounterPRO will attempt to import all matching files.
move_to_pending Sets the default directory for where to move the file while it is being imported. Defaults to Null which means don't move file.
attachment_file_filter Sets the file filter for the standard windows file-open dialog box. Defaults to 'All Files (*.*), *.*'
command_line Specifies command to run prior to importing any files. If set, EncounterPRO will run the specified command line and wait for the spawned process (if any) to terminate. This is useful to run an external program which may or may not leave a file when it terminates.
