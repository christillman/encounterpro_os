Problem: User is unable to scan within encounterPRO

Cause: Not Applicable
Solution: Re-install EPRO. Install EPROLIB 33.4 and EPROLIBNET 33.16. Set job to delete jmj temp files. Ensure temp file exists on fat client. Ensure CPRFILES and Temp on the server has all rights for all users.
Details:

Below are some things to try:

1) Re-install EPRO.
2) Install EPROLIB 33.4 and EPROLIBNET 33.16 (or latest versions).
3) Set up job to delete jmj temp files. See KB article 10133.
4) Ensure temp file exists on fat client.
5) Ensure CPRFILES and Temp on the server has all rights for all users.

Article created on 1/19/2005 by Jennifer.Blalock

Article last updated on 1/19/2005 by Jennifer.Blalock

Symptoms: User cannot scan or post. Below error msg is seen in the Event Viewer - w-scan-main - (clicked) Error getting attachment object.
