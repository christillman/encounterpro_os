Problem: Prescriptions from Document Manager Printing Too Large

Cause:
Solution:

Attached is an example of the PDF documents that EncounterPRO will send to the rx-designated printer when the user wishes to print a prescription rather than send it electronically.

This is a standard PDF documents with a page size of 4.25' x 5.5' and is intended to be printed on quarter sheet paper (standard 8.5' x 11' cut into quarters).

Most printers will attempt to resize the document to fit the paper in the tray.  Since the printer is usually not expecting quarter sheets, a common problem is that the printer attempts to resize our document to fit a whole sheet.

The resolution depends on the printer model and driver versions, but falls into two categories:

1)      Configure the printer to know the correct size of the paper that is actually in the tray

2)      Configure the printer to take it's paper size settings from the document being printed

We have had success with the first on HP printers and with the second on Ricoh printers, however each printer is different and drivers are being updated frequently so if there is a problem then the resolution for any specific printer needs to be determined on site by the customer's IT support.

If the correct settings for printing the quarter sheet prescriptions makes the printer unsuitable for printing other documents and the customer wishes to use that printer for other documents, then the solution is to create two virtual printers both pointing to the same physical printer.

EncounterPRO ships with the free version of Foxit Reader which is used to silently print PDF files from EncounterPRO Server.  No windows shell settings are changed so it will not interfere with the customers' choice for PDF viewer.  We currently ship with version 2.3 but that is subject to change.  More information about Foxit Reader can be obtained from their web site here:  http://www.foxitsoftware.com/
