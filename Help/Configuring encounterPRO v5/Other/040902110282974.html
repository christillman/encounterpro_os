How to: Set a Default Encounter Type for an Exam Room

Solution:

You can set a default encounter type for a specific room so that any encounter started in that room will automatically have the same encounter type. This also adds the Check-in button at the bottom of the room screen so that you can manually check-in the patient without having to go to the Charts tab.

This is used when setting up the Telephone Rooms and Provider Call rooms, but can be useful for other, specialized exam rooms as needed.

  1. Open up Microsoft Access.
    1. Open the database that you are wanting to add rooms too.
  2. Open the o_rooms table.
    1. Locate the row for the desired room by room_id.(ex. EXAM1)
    2. Room _status should be set to OK.
    3. The computer_id should be left blank unless you are wanting to set a coumputer to open with a default room. If this is the case you have to go into the o_computers table and get the computer_id for the computer you want to set.
    4. Office_id is usually set to 0001 unless you have multiple offices. If you have multiple offices you can get the office_id from the c_office table.
    5. Status is set to OK.
    6. Enter the encounter_type name in the default_enc column. Get this name from the c_encounter_type table. Be sure that you type it in exactly as it appears in c_encounter_type.
    7. Repeat a-f to set default encounter types for other rooms.
  3. Go into encounterPRO and verify that the rooms are displaying the way that you intended. Each room with a default encounte type should have a Check-in button at the bottom. Manually check in a patient to confirm that the correct encounter type is used.
