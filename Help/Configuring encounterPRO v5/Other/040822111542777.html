How to: Change the fields at the top of the SOAP screen

Solution:

To link a property to a field at the top of the SOAP screen, go to the o_service_attribute table and follow the instructions below for each of three services:  'CHART,' 'EXAM,' and 'SUPERVISOR.' 

First, you will need to know the user_id.  This will typically be the specialty_id for the practice.  If a specific user wanted to have properties added to the SOAP screen just for him or herself, this would be a personal customization that would not typically be considered to be technical support.  The user_id is entered in the o_service_attribute table in the user_id field. 

Second, enter an 'attribute' called 'property_title_x' where 'x' is an integer between 1 and 7.  If  x = 1, you will populate the field in the top left corner of the SOAP screen.  If you choose, 2, you will populate the field just below that.  And so on. 

The 'value' for this attribute should be a short text description that will display at the top of the SOAP screen.  For example, if you wish to have a field for Weight and Height, you might want to type in the text 'Wgt/Hgt.' 

In the example above, you would have entered the following for a Peds practice in the o_service attribute table:

o_Service_Attribute
service attribute_sequence user_id attribute value
CHART 261 $PEDS property_title_3 Wgt/Hgt
EXAM 255 $PEDS property_title_3 Wgt/Hgt
SUPERVISOR 804 $PEDS property_title_3 Wgt/Hgt

(Note that the attribute_sequence is automatically assigned and will be different in different databases.)

The next step is to link the description (or title) you just added to the desired property in c_property.  To do this, enter an attribute called 'property_x' where 'x' has the same value as was selected in the instruction above.  For example, where x = 3, you will want to enter the attribute ' property_3'  in o_service attribute.  

Next, find the function_name and the property_object of the desired property in c_property.  In the example above, the function_name is 'PATIENT_WGTHGT' and the property_object is 'Patient.'  The value that you enter into o_service attribute is the property_object followed by the property_name.  In this example, the value would be 'Patient PATIENT_WGTHGT.' 

In our example, you would want to add the following to o_service _attribute:

o_Service_Attribute
service attribute_sequence user_id attribute value
CHART 260 $PEDS property_3 Patient PATIENT_WGTHGT
EXAM 254 $PEDS property_3 Patient PATIENT_WGTHGT
SUPERVISOR 803 $PEDS property_3 Patient PATIENT_WGTHGT

 

(Note that the attribute_sequence is automatically assigned and will be different in different databases.)

Refresh EncounterPRO to see the new fields.

 

Be sure to clear any old values in o_service_attribute for the field that you want to configure.  For example, you would not want to add a new property_title_3 of  'Hgt/Wgt' for Peds where there is already a property_3 of 'HIPAA.'  You would want to delete the record property_title_3 'HIPAA' or move it to another position at the top of the SOAP before adding the new property_title_3.
