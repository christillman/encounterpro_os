How to: Configure a Composite Observation to Auto Advance When Entering Results

Purpose

Composite observations can be configured to auto advance to the next simple observation when the user enters a result. This is available only for composite observations formatted as drilldown screens. It can be very useful when the user is entering a large amount of data as it saves the user from needing to click the Next button. This technique can be used for PFSH, ROS, Physical Examination, or other composite, drilldown observation.

Modifying existing composite observations is best done 'on the fly' rather than through the configuration tab, since you can quickly check your work to make sure you have made the modifications as desired.

When to Use this Technique

The observation you want to configure must meet all of the following criteria:

  • Composite observation
  • Drilldown format
  • Each constituent observation must offer a mutually exclusive result set (the user can only select one option from the result set)
  • All constituent observations must be simple observations (the system cannot advance between composite observations)

Before You Begin

You should already be familiar with drilldown screens. See Drilldown Format for details.

Details:

Steps
Follow the steps below (this is best done using a practice or 'test' patient):
Set the results for each constituent observation to mutually exclusive:

  1. Display the desired drill down screen.
  2. Select the Other Options button. A menu of options appears.
  3. Select the Configure button. The Composite Observation Definition screen appears.
  4. Select the bar with the observation name for the first constituent observation. A menu of options appears.
  5. Select the Edit Observation button, the Observation Definition screen appears.
  6. Select the gray Results box. The Results Definition screen appears. The result set must be mutually exclusive; that means that the user will only be allowed to select one of the results for this observation.
    For example: Yes and No, Normal and Abnormal. If the results set allows the user to enter text or a numeric value that observation cannot be set to mutually exclusive and will not be able to auto advance.
  7. Make changes to the results set as needed:
    • If you want to remove a result, select the applicable result on the list at the left and select the Delete button.
    • If you want to change the order in which the results display, select the result you want to reorder, select the Move button, then move the result up or down as needed.
    • If you wish to use a pre-set result set you must first delete all existing results. Then select the Add Result Set button and choose the desired result set from the list.
    • If you want to add a new result, select the New Result button.
  8. If you want to save this set of results for future use in another observation, select the Set Result Set button.
    1. Choose Create a new result set.
    2. Enter the name of this new set and select OK. The name of the new set is added to the bottom of the Result Set list.
    3. Select OK to save the results.
  9. On the Observation Definition screen be sure to select Yes for the Results Mutually Exclusive option in the upper right of the screen.
  10. Select OK to return to the list of constituent observations.
  11. Repeat steps 3 - 8 to set all results sets to mutually exclusive and confirm that the results set is appropriate. The Composite Observations screen should be displayed.

Setting the Auto Advance feature:

  1. From the Composite Observation screen, select the bar for the first constituent observation. A menu of options appears.
  2. Select Attributes. The Observation Attributes screen appears.
  3. Select Move to Next for the On Results Entered: flag.
  4. Select OK to save your changes and return to the Composite Observation Definition screen.
  5. Repeat steps 1 through 4 for each constituent observation.
  6. When finished, select OK to return to the original drilldown data entry screen.
  7. Test your modifications by starting at the top of the drilldown screen and selecting the bar for the first observation. When you select a result, the screen should automatically update to display the next observation.

NOTE: This change will not appear for other encounterPRO users until they have re-started the application.
