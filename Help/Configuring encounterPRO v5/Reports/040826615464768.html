Solution Path: Configuring encounterPRO v5 > Reports

How to: Change which report is called by the sigma button

Solution:

From Mark: 'The sigma button on the patient chart looks for a chart page attribute called 'summary_menu_id'.  If it finds it displays that menu with the auto-singleton behavior.

 

If no summary_menu_id chart page attribute exists, then EncounterPRO attempts to display the encounter summary report defined by the preference_id 'summary_report_id'.  In Configuration>Preferences>User Favorite Preferences (maximize the screen to see this bar)>RPT - Encounter Summary Report. Set at the Global level.

 

If no such preference is found then EncounterPRO will attempt to display the following report_id:  '{330C8DB2-A65C-46C1-A58B-72CE6C5B7DC9}' Report description=Encounter SOAP Note'

 

Details:

In version 2, you had to set the Sigma button in the tables. This meant that all charts had the same attributes, the same report.

In version 4, you had to set the Sigma button in the preferences. You could set this preference at various levels.

In version 5, you can now set the Sigma button from the patient chart and set it for that specific chart.

Steps

  1. Turn on Config mode - before you display the chart.
  2. Display a patient chart. When the chart displays, there is a small Configure Tab button in the upper left above the Encounters list.
  3. Select the Next button. The parameters screen lets you select a Summary Button Menu.
  4. Select the menu you want to use. If the menu has just the one button for the encounter report, the menu itself will not appear - the report will automatically appear.
  5. Select Next. The next screen allows you to select an Encounter Summary Report. This is in here for compatibility purposes. You should not use this parameter in version 5.
  6. Select Finished.
