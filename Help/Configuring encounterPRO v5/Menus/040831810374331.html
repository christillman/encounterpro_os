How to: Add/Update the No Known Allergies button (NKA)

Purpose

It can be useful to have 1-touch progress notes on the Allergies and Current Medications screens so that the clinical staff can document that they have reviewed the information with the patient. Many practices have a 1-Touch NKA button on the Allergies screen already so that the staff can indicate that the patient reports having no known allergies. If the patient already has reported one or more allergies, then that progress note isn't applicable. A second 1-touch button that will document that the patient reports no new allergies can be added. See Screen: How to: Add a No New Allergies button.

This procedure details how to add a new NKA button or update the existing one so that both the NKA and No New buttons display properly.

NOTE: Previous instructions will work for the NKA button, but used the Patient context rather than the Encounter context. Also, by naming the Progress Note Type=Allergy Check, one change to a display script will cause both the NKA and No New Allergies progress notes to be visible. 

Before You Begin

Adding the button for the progress note is only half of the procedure. Editing the display script for the report or screen that you want to show the progress note is required, otherwise you will not see the progress notes.

IMPORTANT: If you are not familiar with editing display scripts, do not attempt to perform this procedure. Contact EHR Product Support for assistance. 

 



Details:

Steps
Follow the steps below:

To add the menu button (this procedure is best done using a Test or Practice patient):

  1. Turn Config Mode On.
  2. Display the Allergies screen.
  3. Select the menu display name. The Menu Properties screen appears.
  4. If a Local Copy does note already exist, select Create Local Copy.
  5. Select Edit Menu.
  6. Select New. The Menu Item Properties screen appears.
  7. Select Service, then Patient. A list of Patient-level services appears.
  8. Select New Note. The parameters screen appears. Enter the following:
    • Context object = Encounter
    • Standard Note Type = Blank
    • Special Note Type = Allergy Check
    • Preconfigured Treatment Titles or Keys = Blank
    • Other Note Title or Key = NKA (This is the title of the note.)
    • Note Text = The patient/parent reports no known allergies. (This is the text of the note.)
  9. Edit the name of the button to reflect it's function. Suggested name: 1-Touch NKA.
  10. Select Finished. The Menu Properties screen reappears.
  11. Select the new button bar on the left, then select the Move button.
  12. Use the arrow keys to re-order the menu buttons as desired.
  13. Select OK. The Menu Properties screen reappears.
  14. Select Finished. The Allergies screen reappears. You may need to refresh the screen to see the new button.

NOTE: Adding the button will NOT automatically make the progress note visible. The display scripts for the desired reports and screens must be edited in order for the progress note to display. If you have already edited the display scripts for the No New Allergies button, then you do not have to perform this step. Be sure to check the button and display script. If you are not familiar with editing display scripts, contact EHR Product Support for assistance.

To edit the display script (this procedure is best done using a Test or Practice patient):

  1. Turn Config Mode On.
  2. Display the screen or preview the report on which you want the new progress note to appear. Suggestions: The Quick View Summary, the SOAP Note report, etc.
  3. Scroll down on the display to locate the Allergies section of the display.
  4. Right click in the Allergies sub-title. The Display Script screen appears with the name of the display script as the title.
  5. If a local copy does not already exist an Edit button appears in the upper right of the screen. Select Edit to create an editable local copy of the display script.
    NOTE: Certain reports and display are no editable from a patient context. If that is the case, the Edit button will not appear on the screen and you will not be able to edit the current display script. Note the exact name of the display script and leave the encounter. Go to the Configuration tab and select Display Scripts. Search for the exact display script name you recorded, then select the bar for that display script and the Edit button. Make the changes as described below, then return to the patient encounter to check your work.
  6. Scroll down to find the highlighted (darkened) command for the Allergies sub-title.
  7. Add the following commands to the display script:
    • Format = fontsize=1-,xbold,margin=0/0/0
    • Encounter Progress
      • Note Type=Allergy Check
      • Check Current Encounter Only=Y
      • Display Stype=Standard
  8. Your script should look much like this:

  9. Select Finished. The viewable screen or report appears.
  10. Go to the Allergies screen and select the 1-Touch No New button you added previously.
  11. Return to the edited display script. The progress note should appear in the Allergies section of the report/screen.
