How to: Modify the Order Vaccine button on Vaccine Tab of Patient Chart

Solution:

The current Vaccine or Immunization Status tab for the Patient chart has a button for Order Vaccine. This button allows the user to order a vaccine treatment. Since this does NOT order the vaccine DX and causes billing issues, it may be desirable to change the Order Vaccine to Order Vaccine DX. This would allow the user to select the Vaccine DX, then the vaccine treatment. Reducing billing problems and the number of click for the ordering provider.

CAUTION: This has to be done in the tables!! 

Details:

1. Go to the o_menu_selection table and filter the menu_context column for "ordervaccine"

2. Note the menu id in the menu_id column.

3. Go to the c_menu table and filter the menu_id for the id noted in step 2.

4. Note the menu description (probably Order Vaccine Button Menu).

5. In encounterPRO, go to Configuration, then Menus.

6. Search for the menu description from step 4.

7. Create a copy of that menu. Note the name that you gave the new menu.

8. Delete the Order Vaccine button and add a new button for Ordering the Vaccine DX. This is a Service called Order Diagnosis, the parameter is Assessment type = Vaccine.

9. Save the new menu.

10. Return to c_menu and refresh. Search for the new menu you created.

11. Note the menu_id for the new menu.

12. Return to the o_menu_selection table and replace the menu id for the Ordervaccine menu conext with the menu id noted in step 11.

13. Close all the tables. Refresh the patient chart and test your new button. (The name of the button didn't change when I did this -- if you figure out how to change it, please update this solution)
