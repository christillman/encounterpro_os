How to: Configure a Report Button to Use Document Manager

Purpose

Whether or not a report is routed to the printer or sent to Document Manager is determined by how the report is ordered. The menu button used to order a report can be configured to use Document Manager.

Before You Begin

There are a few configuration parameters that must be defined when setting up a report button that will use Document Manager.
Document Creation flag

  • Report Only. (default) the report and display/print only. No document is persisted.
  • Order Document. No document is displayed or printed right now, but a Document workplan item is created and associated with the current context. The actual creation/disposition of the document will happen later using the Document Management screen.

Document Purpose
Every document processed by Document Management must have a purpose (within the context: encounter, treatment, patient). The purpose tells encounterPRO what type of document is being processed so that it can make the correct routing decision. For example, if the purpose of the document is NewRX that document can be 'sent' to the patient or to a Pharmacy.  If  Patient is selected then the document may be routed only to a printer.  If instead a pharmacy was selected as the recipient, the document may be routed via fax or SureScripts-RxHub.

Send/Print Now
Documents using Document Manager can still be automatically routed to the printer, if desired. This is set by individual report, so if the practice wants all prescriptions to automatically print immediately, this flag is set to Yes. The default setting is No, which allows a user to determine the routing for an individual instance of a report.

Steps
Follow these steps:

  1. Turn on Config Mode. (See How to: Turn on Configuration Mode.)
  2. Navigate to the screen displaying the menu you want to change. (For example, if you want to change the buttons available on the Current Medications screen, display the Current Medications screen.) Notice the small text next to the menu buttons, this is the system name of the menu.
  3. Select the small text with the name of the menu to access the menu configuration screen. All menus have a JMJ Version that you cannot edit. If you want to change a menu, you must first create a local copy.
  4. Select the Create Local Copy button.

    NOTE: If you later decide that you want to revert to the JMJ Version, select the Local Copy and select Inactivate, then select the JMJ Version and select Activate.

  5. If you already have a Local Copy, be sure that bar is highlighted (darkened), then select the Edit Menu button. The Menu Properties screen appears.
  6. Select New. A selection box appears for you to choose the type of button function.
  7. EncounterPRO reports are a service, select Service. A list of service types appears.
  8. Select the report context for the report:
    • General
    • Patient
    • Encounter
    • Treatment

    A list of services for the level chosen appears.
  9. Select the EPRO Report service.
  10. If there are further parameters to define for this service a screen appears to allow you to enter the desired parameters. Select Finish. The Menu Item Properties screen reappears.
  11. Select Order Document, then Next.
  12. Enter a destination, if desired. (Not required for Document Manager) Select Next.
  13. Enter a printer id or leave the Printer parameter blank to use the default printer. Select Next.
  14. Select the report, then select Next.
  15. Select Next three times until the Print/Send Now parameter appears. If you want the document to automatically print to the selected or default printer, select Yes. If you want to be able to choose the recipient and route, leave the parameter as No.
  16. Select Next three times until the Context Object parameter appears. Select the context of this report.
  17. Select Next.
  18. Select the Document Purpose from the list. This is required for any electronically submitted documents.
  19. Select Next.
  20. If you want to send the copies of this particular report to a folder other than the default set for this report context, select a Folder. Otherwise, select Finished. The Menu Item Properties screen reappears.
  21. At this point, you may want to change the Button Title (this appears on the menu). When done making changes, select Finished. Your new button appears as the last button on the menu.
  22. To reorder the buttons on the menu, select the button, then select the Move button. Use the arrow buttons to move the button into the desired position. When finished moving buttons, select OK, then Finished. The screen reappears with the new button.

Changing the Button Icons

With the Menu Properties displayed, select the button for which you want to change the icon (picture).

  1. Select the Edit button. The Menu Item Properties screen appears.
  2. Type the filename for the new button icon in the Button field.
  3. Select Finished. The Menu Properties screen reappears. A small thumbnail version of the new button icon appears.
  4. Select OK, then Finished to return to the original screen. The new button icon is displayed.


TIP: You need to know the exact system filename of the button icon. If there is another menu that uses the icon you want to use, display the Menu Properties of that menu, then select the bar for the button with the icon you desire and select Edit. The button filename is displayed in the Button field.
