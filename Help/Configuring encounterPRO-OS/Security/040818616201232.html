Description
There are several levels in the encounterPRO security model. Settings at each level can influence which tasks a user can view or perform.

Each level as you go down the pyramid is more specific about which functions are granted or restricted.
Details:

Levels of Security
The first level of security is logging into the encounterPRO system.
User profile
Special Permissions
Another area in the user profile sets the general permissions granted to that user. There are Special Privileges such as:

  • May Access Clinical Data -- set to No if the user should not be allowed to view any clinical data
  • May Prescribe Medications -- set to Yes only for those users who are legally allowed to prescribe medications. If this is set to No, a user will get a warning message when he/she tries to order a prescription medication. The medication can be documented in the chart, but a prescription form will not print.
  • All privileges and access to all services (Super User) -- If a user is designated a 'Super User' then any other restrictions selected under Permissions and Authorize Services are ignored.


EncounterPRO Privileges
A second set of permissions that can further refine the user's access and abilities to perform certain tasks includes:

  • May read audit records -- audit records include user audit logs, encounter audit logs, and assessment and treatment audit logs.
  • May edit common standard exams -- all users can edit their personal standard exams, it may be desirable to restrict the number of users who can edit the common exams.
  • May edit common treatment lists -- all users can edit their personal treatment lists, it may be desirable to restrict the number of users who can edit the common treatment lists.
  • May set or modify computer preferences -- computer-level preferences affect all users who log in to that specific workstation.
  • May use encounterPRO in Configuration Mode -- only users who have been instructed in configuration features should be allowed to use Configuration Mode.
  • May edit procedure charges -- this permission allows a user to change the charges, CPT codes or other billing information for procedures in the database.
  • May edit common short lists -- all users can edit their personal short lists, it may be desirable to restrict the number of users who can edit the common short lists.
  • May edit the visit level and coding for any encounter -- the encounter owner and encounter owner's supervisor can edit the visit level and billing for their encounters, if a billing specialist needs to be able to adjust the billing for an encounter, they need to be granted this privilege.
  • May set or modify global preferences -- global preferences affect every user of the encounterPRO system in your practice. This privilege must be restricted to only those users who have been instructed in encounterPRO configuration.
  • May scribe for other users -- this permission allows a user to document in the patient chart under the user ID of another user, usually a provider.
  • May see the Task List for any user -- if this permission is set to Yes, the user will have an 'Other' tab on the Task list. When the Other tab is selected, he/she can select a user from the practice and view or manage that user's My Task list.
  • May take over any encounter -- Providers can already take over encounters from another provider, this permission allows other clinical staff to be able to take over an encounter from a provider.
  • May set or modify preferences for other users -- this permission should be granted only to those users who have been instructed in encounterPRO configuration.

    All the user profile permissions are initially defaulted to No.

Authorize Services
The Authorize Services level of security allows the Super User or EHR Product Support to restrict or grant access to explicit encounterPRO features on a user by user or role by role basis.  There is a default security for each service that can be either:

  • Allow -- allow all users to perform this service
  • Don't allow -- allow only those users specifically granted access the ability to perform this task. Sensitive tasks, such as Configure Preferences, and Uncancel Encounter are defaulted to Don't Allow.


NOTE: Users with the Special Permission of 'All privileges and access to all services' are considered Super Users and are always granted access to all services in the Authorize Services lists. You do not have to explicitly grant a Super User access to a service with 'Don't Allow' security.

Service Types
The items on the Authorize Services list are at the most granular level. They are divided up by service type:

  • General -- tasks that can be performed on data across the practice, such as Multiple Chart Export and all Configuration tasks
  • Patient -- tasks that are performed on or for a patient, such as checking in the patient, entering current medications, viewing the patient's chart, etc.
  • Encounter -- tasks that are performed during an encounter, such as changing the patient room, approving an encounter, canceling an encounter, etc.
  • Assessment -- tasks that are specific to assessments, such as adding an attachment, rediagnosing, closing the diagnosis, etc.
  • Treatment -- tasks that are specific to treatments, such as associating a treatment with a specific assessment, ordering treatments, refilling prescriptions, closing treatments, etc.
  • Observation -- tasks that can be performed specific to an observation, such as adding a comment, sending a task related to this observation, etc.
  • Attachment -- tasks that can be performed on attachments, such as creating a message or task related to this attachment, creating a document, etc.

How To Customize Security for Your Practice
It is best to take a few minutes in the early stages of implementation to discuss with the EHR Implementation Coordinators the make-up of your staff and get a general idea of what permissions you want to grant to which users and if there are specific areas that you want to further restrict access.

Each practice should have at least one user designated as the Super User. This person will need to attend additional instruction to be able to perform those tasks restricted to the Super Users.

As the practice progresses in using encounterPRO, there may be periodic needs to re-visit the security layout and make changes. Again, your EHR representative can assist you in making those decisions and coordinate educational opportunities.
