How to: Grant and Restrict User Access Using Authorize Services

Purpose
After you have set the user's high-level access privileges on the User Configuration screen, you can refine those privileges using the Authorize Services function. Authorize Services allows the encounterPRO Administrator or Super User to grant and revoke access to specific screens and functions in encounterPRO. An expiration date can even be set for those users who need temporary access.

Before You Begin
Review the information in User Access Rights and Authorize Services.

The default for most services is 'Allow,' meaning that unless otherwise restricted because of the privileges set on the User Configuration screen, all users will be allowed to access the screen or function. For those functions, individual users can be restricted from access. For certain security-sensitive functions the default is 'Don't Allow,' meaning that only Super Users will be allowed access unless a user is specifically granted access. Individual users can be granted special access to those functions.

Functions that default to 'Don't Allow' include:

  • Audit
  • Authorize Services
  • Configure Document Purpose
  • Configure Practice
  • Configure Preferences
  • Configure XML
  • Merge Patients
  • Monitor Results
  • Multi-chart Export
  • Refill Requests
  • Register Prescriber
  • Uncancel Encounter

 Steps

To restrict a user's access to a function
If the default for the function in question is Allow, you can restrict access for individual users by following these steps:
NOTE: You cannot restrict access to any function for a user who is designated a 'Super User' on the User Configuration screen. Super User status overrides any settings on the Authorize Services screen.

  1. From the Configuration tab, select the Authorize Services button. The Services/Privileges screen appears. The service type defaults to Encounter.
  2. Choose the appropriate service type by selecting the Service Type bar. Select the appropriate type from the list: General, Patient, Encounter, or Treatment. The list on the left will change based on the service type selected.
  3. Select the service from the list on the left that you wish to restrict. If the Default Security is Allow the users displayed on the Access List with an 'R' to the right of the name are NOT allowed to access the selected service. (If the Default Security is Don't Allow and you want to allow access, go to the (To allow a user access instructions below.)
  4. Select the New Access Record at the bottom of the screen. The Select User screen appears.
  5. Select the name of the user for whom you want to revoke access to this specific service. The Services/Privileges screen reappears with the name of the user on the access list with an 'R' (Revoke) at the end of the gray bar, indicating that he/she does not have privileges to perform that specific service.
  6. Each service has an expiration period that will automatically change the user's access when that expiration time has been reached. If you want to change the default expiration period, select the Change button and enter the desired expiration period.
  7. If you wish to change privileges for this service, select the name bar of the appropriate user and select either Grant or Revoke from the menu that appears. If you select Grant, the 'R' changes to 'G', indicating that privileges for this service have been granted. If you select Revoke, a 'G' changes to 'R' indicating that privileges have been revoked. Repeat steps 4 through 6 to select additional users for whom to grant or revoke privileges.
  8. To remove a user from the Access List, select the user name and select Delete the user will be removed from the list.
  9. When finished making changes to user access rights, select Exit.

To allow a user access to a function
If the default for the function in question is Don't Allow, you can allow access for individual users by following these steps:

  1. From the Configuration tab, select the Authorize Services button. The Services/Privileges screen appears. The service type defaults to Encounter.
  2. Choose the appropriate service type by selecting the Service Type bar. Select the appropriate type from the list: General, Patient, Encounter, or Treatment. The list on the left will change based on the service type selected.
  3. Select the service from the list on the left that you wish to restrict. If the Default Security is Don't Allow the users displayed on the Access List with an 'G' to the right of the name ARE allowed to access the selected service. (If the Default Security is Allow and you restrict access, go to the (To restrict access instructions above.)
  4. Select the New Access Record at the bottom of the screen. The Select User screen appears.
  5. Select the name of the user for whom you want to allow access to this specific service. The Services/Privileges screen reappears with the name of the user on the access list with an 'G' (Grant) at the end of the gray bar, indicating that he/she has privileges to perform that specific service.
  6. Each service has an expiration period that will automatically change the user's access when that expiration time has been reached. If you want to change the default expiration period, select the Change button and enter the desired expiration period.
  7. If you wish to change privileges for this service, select the name bar of the appropriate user and select either Grant or Revoke from the menu that appears. If you select Grant, the 'R' changes to 'G', indicating that privileges for this service have been granted. If you select Revoke, the 'G' changes to 'R' indicating that privileges have been revoked. Repeat steps 4 through 6 to select additional users for whom to grant or revoke privileges.
  8. To remove a user from the Access List, select the user name and select Delete the user will be removed from the list.
  9. When finished making changes to user access rights, select Exit.
