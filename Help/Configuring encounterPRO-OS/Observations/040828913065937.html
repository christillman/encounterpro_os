Problem: Result entry screens are advancing automatically and I cannot enter all the needed results

Cause:

Composite observations can be configured to automatically advance to the next observation when the user selects a result. This should only be used when the results set for each observation is mutually exclusive; the user can only select one result for that observation.

Solution:

Check the Constituent Observations' Mutually Exclusive Flag:

  1. Display the composite observation and select Configure. The Composite Observation Definition screen appears.
  2. Select the first constituent observation and select Edit Observation. The Observation Definition screen appears.
  3. If the flag is set to Yes for Results Mutually Exclusive, be sure that is appropriate for the results set for the current observation. Mutually exclusive cannot be used with text, numeric, or comment results.
  4. Select OK to return to the Composite Observation Definition screen.
  5. Repeat steps 2 through 4 for each constituent observation.

Check the Constituent Observations' Attributes:

  1. From the Composite Observation Definition screen select the first constituent observation and select Attributes. The Observation Attributes screen appears.
  2. If the flag is set to Move Next for On Results Entered, change it to Don't Move.
  3. Select OK to return to the Composite Observation Definition screen.
  4. Repeat steps 1 through 3 for each constituent observation.
  5. Select OK when finished.
