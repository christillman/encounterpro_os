Description
Each appointment type in your scheduling system will have a corresponding encounter type in EncounterPRO. These are direct encounter types, with the patient physically in the office. Additionally, there are indirect encounter types used to enter information in the patient's chart when the patient is not in the office. Indirect encounter types are used to document telephone calls, past encounters, and entry of records.

NOTE: The encounter visit code is NOT based on the encounter type for the visit, but on the type of assessment selected: Sick or Well.

Details:

Common Encounter Types
The table below lists some common encounter types. Your practice may have a different list depending on customization and your specialty.


Direct Encounter Types

  • Allergy shot only
  • Visit Consult
  • Visit Flu shot only
  • Visit Hospital
  • Discharge Management
  • Initial Hospital Consult
  • New/Established Patient
  • Office Visit (Sick)
  • New/Established Patient Well Visit
  • Newborn Care Visit
  • Nurse Visit

Indirect Encounter Types

  • Entry of records
  • No Show (billed)
  • Provider-initiated telephone
  • Telephone

Mapping Encounter Types to Appointment Types
The encounter type that is selected when the encounter is created controls which workplan is assigned and whether or not the encounter is billable. If the wrong encounter type is selected, then the screens that appear will not be in the desired order and some screens will not be automatically presented to the user. An encounter type is manually selected if the patient is checked in manually; it is automatically selected based on the appointment type mapping if the patient is automatically checked in.

Before your practices is live on EncounterPRO, the integration between your practice management system (PMS) and the EHR will include mapping your current appointments from your billing system to the appropriate encounter types.
Each appointment type must have an encounter type assigned to it, however, you can have multiple appointment types mapped to the same encounter type. For example, the Established Patient Office Visit encounter type will be used for nearly all sick visit appointment types, whether they are a follow-up, new illness, 5-minute or 30-minute visit. The EncounterPRO EHR Implementation Coordinator will assist your practice in mapping the appropriate encounter types to your existing appointment types.

If you later add new appointments types in your PMS, notify EHR Product Support so that they can be mapped to the appropriate encounter types.
