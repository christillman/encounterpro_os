How to: Modify an Existing Yes/No Observation On-the-Fly

Purpose
You can modify the child observations that make up a Yes/No history by removing, adding, or changing the constituent child observations. Remember that each row (question) on a Yes/No history is configured as a simple child observation.

The Yes/No format is commonly used for Review of Systems (ROS) and for observations such as Child Development and Anticipatory Guidance. An example of a Yes/No format screen is shown below.

Before You Begin
The child observations that are included in a Yes/No history can have a maximum of four  results:

  • Up to 4 performs (perform results are either selected or not selected)
  • 1 perform & 3 comment (comment results allow the user to enter text)
  • 2 perform & 2 comments (this is the traditional Yes/No plus a Comment or two)


Important: The order in which you enter the results while configuring  does matter. The results will appear in the order they were entered

Details:

Steps
Follow the steps below:

  1. Display the Yes/No history that you want to modify. Select Other Items, then Configure. The Composite Observation Definition screen appears.
  2. If you want to remove a child observation from the Yes/No history, select the constituent observation from the list and Remove Observation from the menu that appears.
  3. If you want to change the order that the child observations display on the Yes/No history, select a child observation from the list and select the Move option from the menu. Use the arrow buttons to move that child observation up and down the list. Select OK when finished moving that observation.
  4. If you want to change the description (text) or the results for a child observation:

    a.  Select the child observation from the list of constituents.
    b.  Select Edit Observation from the menu that appears. The Observation Definition screen appears.
    c.  Change the Description if needed.
    d.  Change the Perform or Comment results if needed. Remember that the order in which the results are added during configuration determines the order (left to right) they appear on the Yes/No screen.
    e.  Select OK. The Composite Observation screen reappears.

  5. If you want to add a new child observation, select Add Observation. The Select Observation screen appears. If the child observation exists in the database, you can select it from the list. If the child observation does not already exist, you can create it.

    a.  Select New Observation.
    b.  Select Simple. A list of observation types appears.
    c.  Select the appropriate observation type (for a history, you will usually select Subjective).
    d.  Enter the text that you want to appear on the Yes/No history screen in the Description field.
    e.  If you want to enter results that create a toggle button (Yes, No, Discussed, etc.) select Perform as the result type (this is the default).
    f.  Select the Results gray box.
    g.  Select New Result. The Results Definition screen appears.
    h.  Enter the Result name (example: Yes)
    i.  Select whether this result is normal or abnormal in the Result Type
    j.  Be sure that the Result has Value option is No (for a Yes or No type result that is a toggle)
    k.  Select OK.
    l.  Repeat steps g - k for the other perform results for this observation.
    m.  Select OK to return to the Observation Definition screen.
    n.  If you want to create comment results that allow the user to enter text for this observation, select the Perform bar and then select Comment from the list that appears.
    o.  Select the Results gray box. The Results Definition screen appears.
    p.  Select New Result.
    q.  Enter the text you want to appear on the button of the Yes/No screen in the Comment Title field.
    r.  Select OK.
    s.  Repeat steps p - r to add additional comment results. (You can only have a total of 4 results between the perform and comment results).
    t.  Select OK. The Observation Definition screen reappears.
    u.  Select OK. The Select Observation screen reappears.
    v.  Repeat steps a - u to add additional child observations. When finished, select OK to return to the Composite Observation screen.

  6. Select OK. The original Yes/No history screen reappears. You may need to select I'll Be Back and then select the service bar for that history so that the screen will refresh and display your changes.

NOTE: Other users will not see the changes made until they close and re-start EncounterPRO.
