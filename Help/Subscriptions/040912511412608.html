Triage Protocol: Customer Cannot Find Desired Pharmacy for e-RX

Triage Protocol: Triage Protocol: Customer Cannot Find Desired Pharmacy for e-RX
Overview: The practice is using e-Prescribing with SureScripts and the pharmacy that they want to send a RX to is not listed in encounterPRO...

Symptoms:

Explain to the client the following steps (this is in the Solution Center):

  1. If the pharmacy is NOT showing up in encounterPRO, go to the SureScripts website(http://surescripts.com/#find_pharm)  and  search for the pharmacy.
  2. If the pharmacy does not exist in the SureScripts pharmacy directory then the pharmacy will not be in the encounterPRO directory and is not eligible for e-prescribing. The practice can contact the pharmacy and let them know that the practice is using SureScripts and would like to be able to send e-RX to that pharmacy. The pharmacy must sign up for the service with SureScripts.
  3. If the pharmacy does exist in SureScripts, then the customer should get the complete details (full name, address, phone number) and:

    a) Perform a 'By Zip' search in encounterPRO using the pharmacy's zip code from the SureScripts web site.

    b) If still not found then contact encounterPRO Support with the details of the missing pharmacy
Resolution:

When customer support receives an issue with details of a missing pharmacy, they should follow this protocol:

  1. Verify that the pharmacy is missing at the customer site
  2. Verify that the pharmacy does exist on the SureScripts web site
  3. Escalate issue to a technical support person
Comments: SureScripts-registered pharmacies are downloaded to encounterPRO during the nightly maintenance job.
