Concept: How Lab Integration Matches Results to Patients

Description
When the external lab has finished analyzing the specimen and prepared the report, a results file will be retrieved by encounterPRO for interpretation. encounterPRO will attempt to match the file from the lab with a patient in the database and with an outstanding lab/test for that patient. At this point, one of several things will happen:

A Lab Results File is RetrievedNext...Then...
encounterPRO finds a matching patientencounterPRO finds a matching lab order§  The lab results are automatically populated for the lab
§  A ToDo item is sent to the user who ordered the lab to review the lab results
encounterPRO finds a matching patientencounterPRO does not find a matching lab order§  The results file is saved to the To Be Posted list under Utilities, Scan/Post Images 
§  The user must manually post the results file
encounterPRO does not find a matching patient§  The results file is saved to the To Be Posted list under Utilities, Scan/Post Images 
§  The user must manually post the results file

 

Details:

When a Matching Patient and Lab/Test is Found
When encounterPRO finds a matching treatment, based on the specimen ID, the patient name, lab ordered date, and lab ID. The results will be automatically input for that lab and a To Do item will be created to review the lab results. The To Do item can be sent to either the:

  • Workplan Owner (the user who ordered the lab)
  • Primary Provider for the patient
  • Encounter Owner (for the encounter during which the lab was ordered) This is the default.


The default addressee is set in the workplan by JMJ Configuration.

When a Matching Patient or Lab/Test is Not Found
When encounterPRO does not find a matching patient or finds a matching patient but does not find a matching treatment. The results file is placed in the To Be Posted list so that it can be manually posted to the appropriate patient's chart.

Mapping Labs and Results
When you first begin bringing in lab results using the Lab/encounterPRO integration, there will be a number of lab results that cannot find a matching lab/test treatment. This is due to the mapping that needs to be defined for each practice. When no match is found in the mapping between the lab ID from the lab facility (for example, LabCorp) and the treatment in encounterPRO, you will need to select the appropriate lab/test using the manual posting procedure. After you have made that mapping 'connection' once, encounterPRO remembers which lab ID to match to that treatment for the subsequent patients. In many cases, the lab ID from the lab facility will be used for multiple treatments within encounterPRO, for example the many Urinalysis tests available in encounterPRO may all have the same lab ID in LabCorp. EncounterPRO allows you to map one lab ID to multiple encounterPRO treatments. You will need to make the mapping 'connection' for each lab ID/treatment combination once in encounterPRO. EncounterPRO will 'learn' your mappings as you manually post lab results. No set up is required for mapping prior to using the lab integration.
